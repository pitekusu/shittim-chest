# 実装・試験・検証記録

## 1. 目的・記録規約

本書は各implementation sliceで実施した試験と検証の実測証跡を累積する。設計上の合否条件は[[18_試験・品質保証設計]]、実装順序と要求対応は[[19_実装計画・トレーサビリティ]]を正とする。

- sliceごとにID、scope、branch/base、tool version、command、result、coverage、security finding、生成artifact、未実施範囲、既知問題、PR/CI結果を記録する。
- 失敗、再実行、条件付きskipを削除せず、後続結果から参照できる形で残す。
- secret、本番runtime値、user content、Discord/OpenAI本文、credential形式、ローカル絶対home pathを記録しない。
- local合格とGitHub CI、production integration、deploy合格を区別し、未実施を成功扱いしない。
- 本書はpublic-safeなObsidian正本からrepository `docs/`へmirrorする。

### 4.2 Implemented scope

- `.github/workflows/ci.yml`に`quality`、`tests`、`security`、`package`、`docs-public-safety`を実装した。PR、main push、manualで全5 jobを実行し、workflow permissionは`contents: read`だけとする。
- PR workflowはGitHub-hosted `ubuntu-latest`だけを使い、secret、OIDC、Environment、AWS/Discord/OpenAI接続、`pull_request_target`を使用しない。PRの旧runだけをcancelし、main runは固有groupで完走させる。
- checkout v7.0.0、setup-uv v8.3.2、upload-artifact v7.0.1、dependency-review-action v5.0.0を公式releaseのfull commit SHAへ固定した。Dependabotの`github-actions` ecosystemで同一行version commentごと更新する。
- Gitleaks 8.30.1とactionlint 1.7.12は公式release archiveのversionとSHA-256を固定する。実行時latestや追加Action allowlistに依存しない。
- `cyclonedx-python-lib[json-validation] 11.11.0`を直接dev dependencyとし、uv CycloneDX 1.5 preview出力を公式strict JSON Schemaで検査する。root project、`uv.lock`の全PyPI package、duplicate/unknown ref、unsupported sourceもfail closedにする。
- source SBOMをPR/main CI artifactとして30日保存する。release image SBOMとattestationはSTEP-08/10へ分離する。
- Python Dependabot graph jobをGitHub managed inventoryとして採用し、custom Dependency Submissionと`contents: write`を使用しない。`dependency-graph.yml`でDependabotと毎時開始時の混雑を避けた毎週火曜12:17 JST/manualにGitHub SPDX 2.3とstable mainのCycloneDX/`uv.lock`を照合する。SPDX exportにcommit SHAがない制約は比較前後のmain SHA確認で補い、main移動時はworkflowを明示失敗させて再実行する。反映遅延は30秒間隔・最大10回のbounded pollingとする。
- `tools/check_docs.py`で14文書のfile set、UTF-8/LF、frontmatter、calendar date、fence、Wiki file/heading、path traversal、公式資料記録を検査する。private Vaultのbyte一致はlocalだけで検査し、CIへVault accessを与えない。
- package jobは生成wheelが1件であることを確認し、別の一時環境へ明示wheelを`--no-index --only-binary`でinstallする。runtime dependency追加時にwheelhouseが不完全なら安全側に失敗する。source tree外から`-I` importし、version、site-packages path、`py.typed`を検査する。

- Rulesetへ5 required checkをGitHub Actions App source、strict policy付きで追加し、CodeQL High以上のmerge protectionを追加する。`cdk`と`container-arm64`はまだrequiredにしない。
