# 実装・試験・検証記録

## 1. 目的・記録規約

本書は各implementation sliceで実施した試験と検証の実測証跡を累積する。設計上の合否条件は[[18_試験・品質保証設計]]、実装順序と要求対応は[[19_実装計画・トレーサビリティ]]を正とする。

- sliceごとにID、scope、branch/base、tool version、command、result、coverage、security finding、生成artifact、未実施範囲、既知問題、PR/CI結果を記録する。
- 失敗、再実行、条件付きskipを削除せず、後続結果から参照できる形で残す。
- secret、本番runtime値、user content、Discord/OpenAI本文、credential形式、ローカル絶対home pathを記録しない。
- local合格とGitHub CI、production integration、deploy合格を区別し、未実施を成功扱いしない。
- 本書はpublic-safeなObsidian正本からrepository `docs/`へmirrorする。

### 3.2 Scope and behavior

- `.github/dependabot.yml`でrootの`pyproject.toml`と`uv.lock`を対象に、GitHub Dependabot version updatesを有効化する。
- 毎週月曜09:00 JSTに`uv` ecosystemを確認する。
- minor/patch version updateを`python-minor-and-patch`へgroup化し、major updateは個別Pull Requestとして自動mergeせずreviewする。
- version updateの同時open Pull Requestを最大5件とし、`pitekusu`をassigneeにする。
- security updateはversion update groupへ混在させず、GitHub managed security updateの即時経路を維持する。
- 現時点でworkflow、Dockerfile、npm manifestは存在しないため、GitHub Actions、Docker、npm/CDK ecosystemは各manifest導入stepで追加する。
