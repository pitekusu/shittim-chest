# 実装・試験・検証記録

## 1. 目的・記録規約

本書は各implementation sliceで実施した試験と検証の実測証跡を累積する。設計上の合否条件は[[18_試験・品質保証設計]]、実装順序と要求対応は[[19_実装計画・トレーサビリティ]]を正とする。

- sliceごとにID、scope、検証結果、coverage、security finding、未実施範囲、既知問題、PR/CI結果を記録する。
- 失敗、再実行、条件付きskipを削除せず、後続結果から参照できる形で残す。
- secret、本番runtime値、user content、Discord/OpenAI本文、credential形式、ローカル絶対home pathを記録しない。
- local合格とGitHub CI、production integration、deploy合格を区別し、未実施を成功扱いしない。
- 本書はpublic-safeなObsidian正本からrepository `docs/`へmirrorする。

2026-07-20に本書を圧縮した。コマンド逐次実行ログ、check run IDの羅列、個別test case名の列挙、長い差分説明は集約し、未解決事項・残課題・risk acceptanceと外部impactは省略していない。詳細な逐次証跡は各Pull RequestとGitHub Actions runを外部証跡とする。各sliceの最終PR/merge commitはAGENTS.md進捗表も正とする。

external impactの共通注記: 個別記載がない限り、各sliceは実Discord token/Application/Guild、production AWS resource/credential、OpenAI有料API、GitHub本番設定を使用・作成・変更していない。
