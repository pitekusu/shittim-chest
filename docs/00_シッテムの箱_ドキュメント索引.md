## Project overview

Discord上で3つのprivate-configured personaが初回意見、相互参照後の最終案、匿名投票を行い、moderatorが進行、集計、最終決定、記録を担当する単一Guild向けBotを構築する。

詳細設計は2026-07-16時点で完了している。Public repository、community files、Ruleset、Production Environment、managed security settingは構成済みである。STEP-01でPython 3.14.6/uvのproject基盤とdomain state machineを実装した。STEP-02でread-only Public PR CI、CycloneDX source SBOM、GitHub managed Dependency Graph/SPDX照合、package smoke、公開文書検査を実装中である。application use case、外部adapter、container、CDK、AWS resource、Discord Applicationは未実装である。

## Document responsibility・conflict handling

- [[01_要求仕様書・基本設計書]]は「何を満たすか」という要求と外部制約を定義する。
- [[02_議論事項・意思決定記録]]は選択理由、受容risk、廃止案を記録する。
- 各詳細設計書は対応領域の実装可能なinterface、data、failure処理を定義する。
- [[18_試験・品質保証設計]]は合否判定、[[19_実装計画・トレーサビリティ]]は対応関係と実装順序を定義する。

単純な文書優先順位は設けない。矛盾を発見した場合は実装を停止し、ADRへ影響と判断を記録して、要求、意思決定、詳細設計、試験、traceabilityを同じ変更で更新する。

## Source of truth・GitHub mirror

本folderの14文書を公開可能な正本とする。GitHub repositoryの`docs/`はreview・履歴・CI検証用の一方向mirrorであり、直接編集しない。同期sourceは`--source`または`SHITTIM_DOCS_SOURCE`で明示する。本番固有値とpersona内容は本folder外の非公開operator sourceとversion付きSSM parameterへ分離する。

## Deploy-time inputs

- 1件以上のchannel ID allowlist
- AWS account IDとalarm/budget通知先
- 4 Discord Application IDとSecureStringのtoken
- version付きRuntimeConfigと4 PersonaConfigのParameter名
- OpenAI project/API key/model availability

実値は公開mirrorへ、token/API keyはObsidianへ記録しない。詳細は[[19_実装計画・トレーサビリティ#4. Deploy-time required inputs]]を参照する。

## Official-source policy

詳細設計、実装、review、障害対応の各作業turnで最新の公式一次資料へ再アクセスし、URL、確認日、対象version/service、設計への反映を記録する。料金、model、Fargate Spot、package version、Discord API制約、DNS recordは実装日とdeploy日の双方で再確認する。確認不能なら推測で確定しない。

各詳細設計書の「公式資料確認記録」は2026-07-16に確認済み。全体の実装開始条件は[[19_実装計画・トレーサビリティ]]を正とする。
